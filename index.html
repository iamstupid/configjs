configjs

configjs is a powerful tool to make overload functions

Everyone knows JavaScript doesn't allow function overload, so the traditional way to avoid it is to use JavaScript arguments or check argument type. It is unscientific and it makes the code hard to read and understand (just think about you are reading a open source library with if(typeof)s everywhere and function redirects(a famous example is jQuery)) 

//a piece of bad code
function help(){
  if(arguments.length===0){
    print_help_message();
  }
  var k=arguments[0];
  if(typeof k==="object"){
    if(typeof k.help==="undefined"){
      print(helpMap(k));
    }else{
      print(k.help);
    }
  }
  if(typeof k==="function"){
    print(helpFunctionMap(k));
  }
}

The code is unreadable!

Why I should write like this?

The answer is use Object to send parameters in.

function help(a){
  if(a.function){
    print(helpFunctionMap(a.function));return;
  }
  if(a.object){
    print(a.object.help?a.object.help:helpMap(a.object));return;
  }
  if(a.var){
    print(helpScope(a.var,a.module));return;
  }
  print_help_message();
  return;
}

Now the code is clear.
So this is how I can help you to solve this.

argument pattern

//usage
//a tutorial
function add(args){
  var argsp={
    a:[1,false],
    b:[2,false]
  };
  args=testConfig(args,argsp);
  return args.a+args.b;
}
//now how it outputs
add();//3
add({b:4});//5
add({a:3,b:6});//9
add("a=6|b=6");//12
//we simply parse strings using split method,so don't use add("a={a:\"|\"}"),| will split argument list.
